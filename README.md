## Anforderungen

### Dokumentation

- **Wiki**: Enthält die **Idee, Zielstellung, Entwicklungszyklus** und **Reflexion über den Prozess**.
- **UML-Diagramme**: Dokumentation der **Softwarearchitektur** (Klassendiagramme, Sequenzdiagramme etc.).
- **KI-Nutzung**: Explizite Dokumentation aller **KI-Tools** (z. B. Chatbots, Code-Generierung) und deren **Beitrag zum Projekt**.

### Versionsmanagement

- **Branches**: Sinnvolle Benennung (z. B. `feature/...`, `bugfix/...`, `docs/...`).
- **Commits**: Klare, deskriptive Nachrichten (z. B. `"Add user input validation for catalog numbers"`).
- **Pull Requests (PRs)**: Beschreibend und mit Bezug auf die **umgesetzten Features** oder **Bugfixes**.
- **Teamwork**: Regelmäßige Updates und Reviews im Team.

### Implementierung

- **Sprache**: C#.
- **Struktur**: Modularer Aufbau, sauberer Code, Kommentare für komplexe Logik.

## Themenfelder

Wählen Sie ein Thema aus den folgenden Vorschlägen oder schlagen Sie eine **eigene Idee** vor (Genehmigung erforderlich).

- **Textanalyse**: Analyse von Wikipedia-Artikeln (vgl. [Beispiel 1](link_zu_beispiel1)).
- **Raspberry Pi**: Entwicklung einer C#-Anwendung für den Raspberry Pi (Hardware kann gestellt werden).
- **Math.NET zu LaTeX**: Transformation von Math.NET-Inhalten (Matrizen, Formeln) in LaTeX-Code.
- **ML.net**: Anwendung von **ML.net** für KI-basierte Lösungen (z. B. Klassifikation, Regression).
- **Social Media Analyse**: Auswertung von Twitter-Nachrichten (z. B. Stimmungsanalyse).
- **Vorhersagemodelle**: Implementierung von Prognosealgorithmen für Datensätze (Wetter, Börsenkurse etc.) (vgl. [Beispiel 2](link_zu_beispiel2)).
- **Bahn-Daten-Interface**: Entwicklung eines Interfaces für die Daten der Deutschen Bahn ([API-Dokumentation](https://www.dotnetpro.de/update/api/db-api-marketplace-ersetzt-open-api-portal-2769643.html)).
- **Matheaufgaben-Generator**: Erstellung und automatisierte Auswertung von Mathematikaufgaben für Klausuren.
- **GovData-Analyse**: Analyse von Open-Data-Datensätzen (z. B. von [GovData](https://www.govdata.de/web/guest/suchen/-/searchresult/q/Freiberg/f/state%3A14%2C/s/relevance_desc)).
- **OSM-Datenverarbeitung**: Verarbeitung von OpenStreetMap-Daten mit [OSMSharp](http://www.osmsharp.com/).
